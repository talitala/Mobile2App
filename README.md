The Event-Tracking App was designed to help users organize and keep track of upcoming events in a simple and efficient way. The main goal was to reduce missed appointments or forgotten plans by giving users a centralized place to store events, view them clearly, and receive reminders. The app addresses user needs such as time management, organization, and quick access to schedule information without requiring a complex or overwhelming interface.

To support these needs, the app included three main screens: a login screen, a home screen with a grid display of upcoming events, and an event detail screen for adding or editing event information. The login screen also allowed users to create an account, which made access simple and consistent. The home screen was designed with a clean grid layout so users could quickly scan their events, while the event input screen kept data entry straightforward with clear fields for time, date, and event details. These UI decisions kept the experience minimal and intuitive, reducing the number of steps needed to complete tasks. The design was successful because it prioritized clarity and ease of use, making it easy for users to interact with the app without confusion.

When coding the app, I focused on a structured and modular approach. I separated functionality into logical components such as user authentication, event management, and database handling. I also relied on step-by-step testing as I built each feature, making sure each part worked before moving on to the next. Using a database-driven approach helped ensure data was stored and retrieved correctly across screens. These strategies are useful for future projects because they make code easier to debug, scale, and maintain.

Testing was done throughout development by running the app frequently and checking each feature individually. I verified that login worked correctly, events were properly added and displayed, and notifications triggered as expected. This process was important because it helped catch errors early and ensured the app functioned as intended before finalizing it. It also revealed small issues with data flow between screens, which were fixed by adjusting how information was passed and stored.

One challenge during development was ensuring smooth data transfer between screens, especially when updating the event list in real time. I had to adjust how the database was accessed so changes would immediately reflect on the home screen. This required refining the structure of how data was retrieved and updated. I was particularly successful in implementing the event grid display and connecting it properly to the database, as it required both UI design and backend logic working together effectively.
